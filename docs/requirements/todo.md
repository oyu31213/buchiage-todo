# 要件: Todo アプリ（最小版）

## 目的
ログインした本人が、自分のやるべきことを「追加 → 完了チェック → 不要なら削除」で
管理できるようにする。Claude Code の開発フロー（CLAUDE.md・スキル・Plan mode）を
体感するための学習用として、まず最小限で確実に動くものを作る。

## スコープ（やること）
- Clerk によるログイン / ログアウト（メール/パスワード および **Google アカウント**でのログイン）
- ログインした本人のタスクだけを一覧表示する（他人のタスクは見えない）
- タスクの追加（以下の情報を入力できる）
  - タスク名（必須）
  - 期限（締切日。任意）
  - 優先度（高 / 中 / 低。任意。未指定は「中」）
- タスクの完了チェック（完了 / 未完了を切り替えられる）
- タスクの削除

## スコープ外（やらないこと）
- タスクの**編集**（一度作ったタスク名・期限・優先度のあとからの書き換え）→ 今回はやらない
- タスクの**並べ替え**（ドラッグでの順番入れ替え）→ 今回はやらない
- 表示の**絞り込み / フィルタ**（「未完了だけ」「完了だけ」表示の切り替え）→ 今回はやらない
- タグ付け・カテゴリ分け → 今回はやらない
- 他人とのタスク共有 / 割り当て → 今回はやらない
- 期限が近いタスクの通知・リマインド → 今回はやらない
- 検索 → 今回はやらない

## 受け入れ条件（= 完成の定義 / テストと最終確認はこれを使う）

### ログイン
- [ ] 未ログインの状態でアプリのタスク画面を開こうとすると、ログイン画面に移動する
- [ ] ログイン画面に「Google でログイン」ボタンが表示される
- [ ] 「Google でログイン」ボタンを押すと Google のログイン画面に進み、許可するとアプリにログインできる
- [ ] ログインに成功すると（メール/パスワード・Google のどちらでも）、自分のタスク一覧画面が表示される
- [ ] ログアウトボタンを押すと未ログイン状態になり、タスク画面は見られなくなる
- [ ] 別のユーザーでログインすると、そのユーザー自身のタスクだけが表示され、他人のタスクは表示されない

### タスクの追加
- [ ] タスク名を入力して「追加」ボタンを押すと、入力したタスクが一覧の一番上に表示される
- [ ] 追加時に期限（締切日）を指定すると、そのタスクに期限が表示される
- [ ] 追加時に優先度（高 / 中 / 低）を指定すると、そのタスクに優先度が表示される
- [ ] 優先度を指定せずに追加したタスクは「中」として表示される
- [ ] タスク名が未入力のまま「追加」ボタンを押すと「タスクを入力してください」と表示され、タスクは追加されない
- [ ] 追加した内容は、ページを再読み込みしても消えずに残っている

### 完了チェック
- [ ] タスクのチェックを押すと、そのタスクが「完了」の見た目（例：チェック済み・取り消し線）に変わる
- [ ] 完了済みタスクのチェックをもう一度押すと、「未完了」の見た目に戻る
- [ ] 完了 / 未完了の状態は、ページを再読み込みしても保持されている
- [ ] 完了にしたタスクも一覧から消えず、そのまま表示され続ける

### 削除
- [ ] タスクの削除ボタンを押すと、そのタスクが一覧から消える
- [ ] 削除した内容は、ページを再読み込みしても元に戻らない（消えたままになる）

## 確定した方針
- 一覧の並び順は「新しく追加したタスクが一番上」で固定（並べ替え機能は今回なし）。
- 完了したタスクも一覧に残す（チェックしても一覧から消えない）。

## デザイン
- ファイル: `docs/design/pencil-new.pen`（Pencil で開いて確認）
- 画面（4状態）:
  1. **ログイン** — アプリ名・「Google でログイン」ボタン・「または」区切り・メール/パスワード入力・ログインボタン・新規登録リンク（実体は Clerk が提供）
  2. **タスク一覧（通常）** — ヘッダー（アプリ名＋ログアウト）／追加フォーム（タスク名・期限・優先度・追加ボタン）／タスク行（チェックボックス・タスク名・期限・優先度バッジ・削除）。完了タスクは青チェック＋取り消し線で残る
  3. **タスク一覧（空）** — 0件のとき「まだタスクがありません」の空状態を表示
  4. **タスク一覧（エラー）** — タスク名未入力で追加した場合、入力枠が赤くなり「タスクを入力してください」を表示
- テーマ色: **黄色（アンバー）ベース**。背景はあたたかい黄色系（#FFFBEB）、ボタン・アクセント・完了チェックはアンバー（#F59E0B、文字は濃色）、リンクは #B45309
- 優先度バッジの色: 高=赤 / 中=黄 / 低=緑
- これらは「見た目の受け入れ条件」として実装・最終確認の基準にする
- 実装時は shadcn/ui の素の部品（`components/ui/`）を土台に、`components/<feature>/` でこのデザインに合わせて組む
